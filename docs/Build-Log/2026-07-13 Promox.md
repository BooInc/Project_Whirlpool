\---



\## Issues Encountered



\### Enterprise Repository



The default Enterprise repository prevented updates because no subscription was installed.



\*\*Resolution\*\*



Disabled the Enterprise repository.



Enabled the No-Subscription repository.



Successfully updated the system.



\---



\## Validation



\- Web interface accessible

\- Repository configuration correct

\- System fully updated

\- No storage errors

\- LVM-Thin storage available



\---



\## Lessons Learned



\- Home labs should use the No-Subscription repository.

\- Kernel updates require a reboot.

\- Proxmox installation is straightforward once repositories are configured correctly.
